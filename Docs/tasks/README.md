# Task Brief Format

Use one file per task under `Docs/tasks/`.

Each task brief should include YAML frontmatter with:

- `id`
- `title`
- `subsystem`
- `priority`
- `owner`
- `depends_on`
- `owned_paths`
- `source_docs`
- `success_gate`
- `report_path`

Required sections:

- `Objective`
- `Why Now`
- `In Scope`
- `Out Of Scope`
- `Required Changes`
- `Verification Required`
- `Success Definition`
- `Open Risks`

The coordinator owns task briefs.
